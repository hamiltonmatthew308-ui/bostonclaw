## 要点

### 1. 差异化价值：从“知晓”到“执行”的最后一公里
网页端 LLM 是“智库”，桌面 Agent 是“手脚”。
- **场景 A：医美设备参数批量处理**。员工有 50 份 PDF 说明书需要提取特定激光频率参数并填入 Excel。网页端需要用户手动上传 50 次，而桌面 Agent 可以通过本地文件权限一键扫描目录、提取并直接操作本地 Excel 进程。
- **场景 B：敏感数据合规性**。医美公司涉及大量客户肖像和诊疗数据，网页端上传存在泄露风险。桌面 Agent 在本地 Runtime 运行，配合本地知识库（RAG），实现了“数据不出域，AI 进桌面”的硬需求。
- **场景 C：多工具协同**。网页端无法感知你正在运行的 ERP 或 CRM 软件。桌面 Agent 可以作为“系统级插件”，在用户操作复杂软件时提供实时上下文辅助。

### 2. 大赛评委视角：排名前 5% 的潜力股
- **最大得分点（经济价值 + 落地效果）**：标准化。43 支队伍里大部分可能在做“XX 业务问答机器人”，那是点状的。Lobster 做的是“分发平台”，是面状的。它解决了“如何让剩下 1499 人也能用上 AI”的基建问题，这在管理者眼中是极具规模化价值的。
- **最大失分点（汇报清晰度）**：如果 Ludviq 讲太多 Zustand、IPC、Electron，评委（非技术负责人）会听睡着。如果不能用一句话说清“为什么不用网页版”，项目会被打上“炫技/画蛇添足”的标签。
- **预估排名**：如果能讲清“基建赋能”逻辑，稳居 Top 3；如果讲成“技术重构”，可能在前 15。
