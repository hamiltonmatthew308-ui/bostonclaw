## 身份
agent-id: gpt | model: gpt-5.2 (Codex CLI)

## 立场
补充：桌面 Agent 的价值不在“能不能回答问题”，而在“能不能进入工作流并被标准化复制”。但如果拿不出 1-2 个制造业硬场景的闭环指标，项目会被评委看成“又一个壳/又一个安装器”。

## 要点
1. 网页端 LLM 擅长“对话式一次性产出”，桌面 Agent 擅长“可编排的重复性工作流”。差异的胜负手是：数据获取与工具执行是否能自动化、是否能规模化复制给非技术员工。
2. 对 1500 人制造业公司，桌面 Agent 的真价值场景应集中在“文件密集 + 流程固定 + 需要产出结构化结果 + 需要追踪版本/责任”的工作：质检/采购/法务/售后/研发文档，而不是泛用问答。
3. 比赛评委是非技术负责人：得分点是“省了多少人时、减少了多少返工/漏项、落地覆盖了哪些部门”；失分点是“看起来像 FOMO 工具推广、没有业务闭环数据、装了但没用”。
4. 如果继续做 Lobster，定位必须从“OpenClaw 社区/社交”收敛为“企业 AI 同事的发行与落地管道”：模板包 + 安装载荷 + 接入网关 + 运营看板。否则价值叙事会漂。

## 建议
- **回答 Q1（差异化）时，别抽象说权限**。直接给评委可感知的 3 个硬例子（都能用 10 分钟 demo 演出来）：
  - 质检：从一批 PDF/图片/表格里抽取不良项 → 自动生成《质检报告》+ CAPA 行动项（责任人/截止日期）+ 复检清单。网页端能做“写报告”，但做不到“自动遍历文件夹/命名规则/版本归档/产出多份标准模板”。
  - 采购：从供应商报价邮件/附件里抽取关键字段（交期/税率/付款/质保）→ 自动形成比价表（Excel/CSV）→ 输出谈判要点与风险条款清单。网页端能总结，但做不到“自动落地到公司模板 + 批处理 + 生成可用表格”。
  - 法务/售后：把合同/售后工单批量归档分类 → 提取条款偏离点/高风险点 → 生成审批摘要并回写到固定格式。网页端能读一份文件，但桌面 Agent 才能做“批量 + 规则 + 可追踪输出”。
- **回答 Q2（比赛）时建议把作品压成一个指标链**：
  - 输入：模板被克隆次数、活跃人数
  - 过程：每周节省人时（从工单/流程中抽样估算也行）
  - 输出：报告质量一致性（漏项减少）、响应时延下降
  - 成本：统一 gateway + 低价模型路由（用数据讲，不要只讲“省钱”）
- **回答 Q3（怎么做）**：我会立刻做一个“部门级 pilot 闭环”，而不是全公司推广叙事：
  - 选一个部门（建议质检/采购/售后三选一）+ 10 人试用
  - 只做 2 个模板包（一个高频，一个高价值）
  - 把安装器变成“可分发的真安装器”（带 payload，不依赖现场下载）并给出卸载/回滚说明
  - 用 7 天拿到第一份“节省人时/减少返工”的数字，比赛就有抓手

## 理由
网页端 LLM 与桌面 Agent 的差异不是能力上限，而是组织落地的“摩擦系数”。制造业里最贵的不是写作本身，而是资料收集、格式一致、批处理、可追踪与协作交付。只要 Lobster 能把这类工作做成模板化的一键闭环，并在一个部门拿到可量化收益，它在 43 支队伍里就会非常靠前；反之如果停留在“工具推广/社交平台/安装向导”，评委会认为价值不够硬、落地证据不足。
