## 要点

### 1. 桌面 AI Agent vs 网页端 LLM——这家医美公司的真实场景

说实话，我绞尽脑汁想了 2 小时，没有找到一个说服力足够的"必须用桌面 Agent"的场景。

**我尝试找的场景：**
- **文件批量操作**：处理招投标文档、技术规格书。但 Google Docs + Gemini Live / ChatGPT 都能做，网页端 API 也能。
- **CRM/ERP 操作**：OpenClaw 操作桌面软件。但医美设备的 CRM 有没有 API？好不好集成？不知道。
- **CAD 图纸批量处理**：本地文件操作 + AI 理解。但 CAD 软件本身有插件生态。
- **邮件/日历/飞书集成**：OpenClaw 能做，但这不是桌面 Agent 特有的能力，网页端 MCP 也能做。

**结论**：如果 Ludviq 想不出来一个具体到"王工在采购部每天重复操作 40 分钟"的场景，那这个项目的立项依据确实有问题。

### 2. AI 大赛评分预测

**我的判断：中等偏下**

得分点：
- 技术概念有一定复杂度，评委可能觉得"有点东西"
- 有具体的可演示界面（如果是 Web 版本的话）

失分点：
- **商业价值说不清**：30% 经济价值，但 OpenClaw 能给医美公司省多少钱？说不上来。
- **落地效果存疑**：老板推了半年没人用，说明这东西对内部用户没有自然吸引力。
- **技术差异化弱**：43 支队伍里，做 AI + 业务结合的肯定有，做纯技术炫技的也有。OpenClaw 的包装不够惊艳。
- **汇报可能吃亏**：Ludviq 自己都怀疑这项目，台上讲起来信心会不够。

**排名预估**：20-30 名之间。能进复赛，但拿奖很难。

### 3. 如果我是 Ludviq

我会**换方向**，但保留核心积累。

**不继续做 OpenClaw 推广的理由**：
- 桌面 Agent 这条路在 2026 年已经很明显是过渡形态。浏览器 MCP 扩展、网页端 Agent 越来越强，桌面 App 的护城河在快速填平。
- 公司 1500 人，老板推了半年推不动——这不是执行问题，是产品本身没有PMF的信号。

**值得保留的积累**：
- AI + 医美设备结合的想法
- Prompt 工程经验
- 对公司业务场景的了解

**应该做的方向**（纯个人建议）：
医美设备 + AI 的真正机会在**临床决策支持**和**营销内容生成**，不是桌面 Agent。
- 临床端：AI 辅助诊断报告生成、治疗方案推荐（需 FDA/NMPA 认证，门槛高但护城河高）
- 营销端：AI 生成案例对比图、客户问答话术、培训内容自动化（风险低，立刻能用）

如果 Ludviq 想赢比赛：**做一个医美诊所的 AI 助手**，能根据客户皮肤状况推荐项目、能生成治疗前后对比的营销素材、能自动回复常见咨询问题——这个评委容易懂，商业价值也说得上来。
