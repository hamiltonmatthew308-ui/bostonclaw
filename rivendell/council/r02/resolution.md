# Resolution — r02

## 议题
这个项目的本质是什么？桌面 Agent 的差异化价值？比赛竞争力？下一步方向？

## 共识

1. **"1500 人不用 OpenClaw 不是安装难，是没场景"**——全员认同（GPT/Gemini/GLM/Kimi/MiniMax 均提及），这是本轮最重要的共识。Installer 解决的是一个不存在的问题——或者说，不是一个有足够规模的问题。

2. **当前叙事（"OpenClaw 推广平台"）在比赛中竞争力不足**——排名预估：
   - Gemini（最乐观）：Top 3（前提是叙事正确）
   - GPT：取决于能否拿出闭环指标
   - GLM：15-25（当前叙事），5-10（换叙事）
   - Kimi：20-30
   - MiniMax：20-30
   - **中位预估：15-25 名，拿参与奖（3000 元）概率高，拿奖（1-3 万）概率低**

3. **桌面 Agent 的真实差异化在于：批量本地文件操作 + 离线运行 + 预装行业知识库**——而不是"能对话"（网页端也能）

4. **叙事能力比代码完成度更决定比赛排名**——评委是非技术部门负责人，10 分钟汇报前 90 秒定生死

## 决议

**采纳 GLM 方案：不换项目，不换代码，换叙事 + 换模板内容。**

理由：
- Kimi/MiniMax 的"换方向"在分析上正确，但 11 天内从零做新项目风险过高
- GPT 的"部门 pilot"方法论正确，但选的场景（质检/采购）Ludviq 不熟
- GLM 抓住了一个所有人都忽略的事实：Ludviq 本人就在做海外参展工作，这是唯一能在 11 天内提供"真实落地证据"的场景
- Gemini 的"AI 员工入职中心"命名有启发但过于通用，"出海 AI 装备包"更精准
