# 演示录屏指南

## 准备工作

1. 在仓库根目录运行 `./scripts/demo.sh`
2. 确认三个入口都正常：
   - Web: `http://127.0.0.1:5173`
   - Server: `http://127.0.0.1:3888/health`
   - Installer: Electron 窗口已打开
3. 打开录屏工具，建议分辨率 `1920×1080`

## 演示节奏

总时长建议控制在 `3-5 分钟`。

### 第一幕：问题定义（30 秒）

- 打开 Web 首页
- 停留在 Hero 区域
- 旁白重点：
  - 企业推 AI，不缺模型，缺的是 adoption 路径
  - 员工常见问题：不知道装什么、不会配、装完不会用

### 第二幕：模板中心（45 秒）

- 滚动到模板库
- 切换几个分类
- 展示不同部门的 AI 同事模板
- 点击“复制安装码”
- 旁白重点：
  - 模板不是 prompt 文本，而是可导入的 AI 同事包
  - 先从角色开始，而不是从空白输入框开始

### 第三幕：安装桥（90 秒）

- 切换到 Installer
- 逐步演示：
  1. 环境准备
  2. 使用场景
  3. 模板中心
  4. 模型接入
  5. 安装执行
  6. 完成交付
- 推荐路线：
  - 场景：桌面个人使用
  - 模板：会议纪要员
  - Provider：任一可用测试 key
- 旁白重点：
  - Installer 帮用户做复杂判断
  - Demo Mode 下安装流程完整可见，录屏稳定

### 第四幕：组织雷达（30 秒）

- 回到 Web 的 Community Radar
- 展示：
  - 在线 Agent
  - 活跃部门
  - 24h 动态
- 旁白重点：
  - 管理者可以看到 adoption 是否正在发生
  - 这不是单个聊天工具，而是组织级采用入口

### 第五幕：收尾（15 秒）

- 回到首页或停在 Installer 完成页
- 旁白重点：
  - Lobster 不做最强 Runtime
  - Lobster 做企业最容易落地的 AI adoption 路径

## 录屏注意事项

- 演示时优先使用 `Demo Mode`，避免真实安装过程不可控
- 如果要展示社区数据，确保 server 已经是当前版本，内置 6 个 Agent 和 25 条动态
- 录屏前先完整跑一遍脚本，确认窗口位置和浏览器缩放
