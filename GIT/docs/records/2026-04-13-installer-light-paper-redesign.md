# Lobster Installer 浅色纸感重构施工档案

- 日期：2026-04-13
- 记录类型：施工档案 / UI 重构记录
- 适用范围：
  - `apps/openclaw-installer/src/renderer`

## 1. 本轮目标

本轮目标不是继续增加功能，而是把 installer 从“能跑的骨架”提升到“有明确审美方向的产品界面”。

用户已经明确否定了上一版偏深色顾问台的视觉效果，因此这一轮重新定调为：

“浅色纸感的安装手册 + 温和的技术顾问”

核心目标：

1. 摆脱深色后台和通用 SaaS 工具感
2. 强化安装顾问的可信度和可读性
3. 让当前向导更像一本带判断的工作手册，而不是一堆卡片和按钮

## 2. 采用的设计方向

本轮明确采用以下视觉判断：

1. 主底色改为暖米白和纸张色，而不是深灰黑
2. 主强调色采用铜橙 / 焦糖棕，而不是高饱和霓虹色
3. 标题采用更偏编辑感的 serif 字体组合
4. 内容主体保留工具界面的效率，但整体气质更接近“高级操作手册”

对应的界面定位是：

“不是营销页，不是控制台，而是一个会帮你做判断的技术手册。”

## 3. 本轮改动范围

### 3.1 外层壳与步骤容器

更新：

- `apps/openclaw-installer/src/renderer/components/WizardShell.tsx`

主要变化：

1. 整体面板改为浅色纸张质感
2. 左侧导航从深色 sidebar 改为目录式章节结构
3. 顶部品牌块和进度条统一到铜橙系
4. 当前步骤、已完成步骤和未开始步骤的状态对比更明确

### 3.2 内容区组件样式

更新：

- `apps/openclaw-installer/src/renderer/App.tsx`

主要变化：

1. `SectionIntro` 改为更强的编辑标题层级
2. `RecommendationBanner` 改为浅底判断条，而不是深色提示框
3. `ChoiceCard` 改为浅色方案卡，降低压迫感并提升阅读性
4. `InfoPanel` 改为手册式说明块
5. 底部主操作按钮、状态提示和导入结果统一到新主题

### 3.3 全局样式

更新：

- `apps/openclaw-installer/src/renderer/styles/globals.css`

主要变化：

1. 全局颜色变量切换为浅色纸感主题
2. `body` 背景改为暖米白渐变
3. 增加轻微网格纹理，强化“工作手册 / 纸面”氛围
4. 调整滚动条和字体栈，使整体质感更统一

## 4. 为什么不继续沿用上一版深色顾问台

上一版虽然已经从原始 demo 状态提升到了“有设计意图的工具界面”，但仍然存在几个问题：

1. 信息密度太高
2. 右侧内容区依然偏工程后台
3. 深色主题让“安装指导”显得过重
4. 左侧有气质，但右侧没有真正跟上

因此，这一轮没有做局部打补丁，而是直接换了整体视觉立场。

## 5. 当前仍保留的产品判断

这次只改了 UI，没有改变功能层的结构判断：

1. 安装码仍然是主导入路径
2. 模板包仍然是内部协议
3. 本地 agent home 仍然是落地真相
4. OpenClaw 仍然只是当前首个 adapter

也就是说：

“换的是呈现方式，不是产品骨架。”

## 6. 当前界面想传达的感觉

本轮希望用户打开 installer 时感受到的是：

1. 这不是另一个复杂工具，而是一个会替我做判断的顾问
2. 这个产品在帮我减少安装和配置决策
3. 它看起来像一套成熟方法，而不是临时拼出来的 demo

## 7. 后续还可以继续精修的点

本轮完成后，后续还可以继续打磨：

1. 字体系统进一步统一
2. 安装码输入区做成更强的主操作区域
3. 结果页强化“AI 同事已就位”的交付感
4. 减少某些步骤页里并排卡片过多造成的阅读压力

## 8. 本轮补充的可访问性状态层

在主视觉完成后，又补了一层状态可读性：

1. 全局交互元素增加了统一的 `focus-visible` 外框
2. 选择卡和步骤项增加了显式状态语义，不只依赖颜色
3. 禁用态按钮保留了视觉退化和不可交互提示
4. 当前步骤在侧栏中有了更直接的非颜色提示

这一步的目的不是继续美化，而是确保纸感视觉在键盘导航和状态识别上仍然清晰。

## 9. 当前结论

这轮 UI 重构之后，Lobster installer 已经不再是简单的 Electron 向导壳，而开始具备清晰的产品气质：

“一份浅色纸感的安装手册，一个温和但有判断力的技术顾问。”
